Encapsulation Example

public class BankAccount
{
    // Private - can't access directly from outside
    private decimal _balance;
    
    // Public property - controlled access
    public decimal Balance => _balance;
    
    // Public method - controls what can be done
    public void Deposit(decimal amount)
    {
        if (amount <= 0)
            throw new ArgumentException("Must be positive");
        
        _balance += amount;
    }
}
✨ Pro Tip

Always mark fields as private and provide public methods or properties. This prevents invalid states like negative bank balances!

Interfaces: The Contract

// Define what methods a class MUST have
public interface IPayable
{
    decimal CalculatePay();
    void ProcessPayment(decimal amount);
}

// Any class that implements this MUST have these methods
public class PayPalPayment : IPayable
{
    public decimal CalculatePay() { return 100m; }
    public void ProcessPayment(decimal amount) { /* PayPal logic */ }
}

Collections & LINQ

Working with lists of data
📋

Common Collections

List<T>

Dynamic array, most common

var names = new List<string>
{
    "John", "Jane", "Bob"
};

Dictionary<K,V>

Key-value pairs

var ages = new Dictionary<string, int>
{
    ["John"] = 25,
    ["Jane"] = 30
};

HashSet<T>

Unique values only

var unique = new HashSet<int>
{ 1, 2, 3, 1 };
// Only {1, 2, 3}
🔍

LINQ: Querying Made Easy

var users = new List<User>
{
    new User{ Name = "John", Age = 25, IsActive = true },
    new User{ Name = "Jane", Age = 17, IsActive = true },
    new User{ Name = "Bob", Age = 30, IsActive = false }
};

// LINQ queries
var adults = users.Where(u => u.Age >= 18).ToList();
var names = users.Select(u => u.Name).ToList();
var active = users.Where(u => u.IsActive).Count();
🎯 Exercise: Practice LINQ

Try these LINQ operations:

  • Filter: users.Where(u => u.Age > 20)
  • Select: users.Select(u => u.Name)
  • Order: users.OrderBy(u => u.Age)
  • First: users.FirstOrDefault(u => u.Name == "John")
