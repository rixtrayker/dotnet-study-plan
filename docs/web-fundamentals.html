How the Web Works

The foundation every developer needs
Week 1 Everyone
🌐

The Request-Response Cycle

Every web interaction follows this pattern

1

User types URL or clicks a link

Browser creates an HTTP request to the server.

2

Server receives the request

Your application determines what to do based on the URL.

3

Server processes and responds

Returns HTML (pages) or JSON (APIs) to the browser.

💡 Think of it like a restaurant

Browser = Customer ordering food | Server = Waiter taking order | Kitchen = Database preparing ingredients | Food = Response

🔗

MVC Pattern

The architecture behind most web frameworks
Week 2 Everyone
🏛️

MVC: Model - View - Controller

Separation of concerns made simple

📝 Controller

Receives user input, calls the right methods, returns responses.

📦 Model

Your data, business rules, and database logic.

🎨 View

What the user sees - HTML, CSS, JavaScript.

📝 Controller

Waiter takes your order and coordinates between you and the kitchen.

📦 Model

Kitchen prepares the food using recipes (business logic) and ingredients (data).

💡 Why MVC?

Separation of Concerns: Each part has ONE job. Controllers handle requests, Models handle data, Views handle presentation. Change your UI without touching your business logic.
