🏛️

MVC: Model - View - Controller

Separation of concerns made simple

flowchart LR subgraph mvc["MVC Pattern"] C["📝 Controller
Handles requests

Business logic
Data access"] V["🎨 View
HTML output

📝 Controller

Receives user input, calls the right methods, returns responses.

📦 Model

Your data, business rules, and database logic.

🎨 View

What the user sees - HTML, CSS, JavaScript.

💡 Why MVC?

Separation of Concerns: Each part has ONE job. Controllers handle requests, Models handle data, Views handle presentation. Change your UI without touching your business logic.

Encapsulation Example

public class BankAccount
{
    // Private - can't access directly from outside
    private decimal _balance;
    
    // Public property - controlled access
    public decimal Balance => _balance;
    
    // Public method - controls what can be done
    public void Deposit(decimal amount)
    {
        if (amount <= 0)
            throw new ArgumentException("Must be positive");
        
        _balance += amount;
    }
}
✨ Pro Tip

Always mark fields as private and provide public methods or properties. This prevents invalid states like negative bank balances!

MVC in ASP.NET Core

How everything connects
🔄

Request Flow in ASP.NET MVC

sequenceDiagram participant Browser participant Routing participant Controller participant Model participant View Browser->>Routing: GET /Users Routing->>Controller: UsersController.Index() Controller->>Model: GetUsers() Model-->>Controller: List Controller->>View: return View(users) View-->>Browser: HTML Page
📝

Controller Example

public class UsersController : Controller
{
    private readonly IUserService _users;
    
    // Constructor injection
    public UsersController(IUserService users)
    {
        _users = users;
    }
    
    // GET /Users
    [HttpGet]
    public async Task<IActionResult> Index()
    {
        var users = await _users.GetAllAsync();
        return View(users);
    }
    
    // GET /Users/Details/5
    [HttpGet("Details/{id}")]
    public async Task<IActionResult> Details(int id)
    {
        var user = await _users.GetByIdAsync(id);
        if (user == null) return NotFound();
        return View(user);
    }
}

// Login endpoint
[HttpPost("login")]
public  IActionResult Login(LoginDto dto)
{
    var user = _context.Users
        .FirstOrDefault(u => u.Email == dto.Email);
    
    if (user == null || !BCrypt.Verify(dto.Password, user.PasswordHash))
        return Unauthorized();
    
    var token = _tokenService.GenerateToken(user);
    return Ok(new { token });
}

// Protected endpoint
[Authorize]
[HttpGet("profile")]
public ActionResult<UserDto> GetProfile()
{
    var userId = int.Parse(User.FindFirst("sub")!.Value);
    return Ok(await _users.GetByIdAsync(userId));
}
⚠️ Security Checklist
  • Always hash passwords with BCrypt or Argon2
  • Store JWTs securely (HttpOnly cookies for web apps)
  • Use HTTPS in production
  • Set appropriate token expiration times
  • Validate all user input

How Middleware Works

Each middleware can process requests and responses

flowchart LR A["📥 Request"] -->|"1"| B["📋 Logging"] B -->|"2"| C["🔐 Authentication"] C -->|"3"| D["⚡ Routing"] D -->|"4"| E["🎯 Endpoint"] E -->|"Response"| D --> C --> B --> A style A fill:#58a6ff,color:#fff style B fill:#d29922,color:#fff style C fill:#f85149,color:#fff style D fill:#a371f7,color:#fff style E fill:#3fb950,color:#fff 
// Custom Middleware
public class RequestTimingMiddleware
{
    private readonly RequestDelegate _next;
    
    public RequestTimingMiddleware(RequestDelegate next)
    {
        _next = next;
    }
    
    public async Task InvokeAsync(HttpContext context)
    {
        var stopwatch = Stopwatch.StartNew();
        
        await _next(context); // Call next middleware
        
        stopwatch.Stop();
        context.Response.Headers["X-Response-Time"] 
            = $"{stopwatch.ElapsedMilliseconds}ms";
    }
}

// Register in Program.cs
var app = builder.Build();
app.UseMiddleware<RequestTimingMiddleware>();
app.UseAuthentication();
app.UseAuthorization();
app.MapControllerRoute(...);
app.Run();

The DI Container

Automatic dependency resolution and lifetime management

sequenceDiagram participant Controller participant Container participant Service Container->>Service: new UserService(repo, mailer) Container->>Controller: inject UserService Controller->>Service: doSomething() Service-->>Controller: result 
// Define services
public interface IUserService
{
    Task<User> GetByIdAsync(int id);
}

public class UserService : IUserService
{
    private readonly AppDbContext _context;
    
    public UserService(AppDbContext context)
    {
        _context = context;
    }
    
    public async Task<User> GetByIdAsync(int id)
        => await _context.Users.FindAsync(id);
}

// Register in Program.cs
builder.Services.AddScoped<IUserService, UserService>();

// Use in controller
public class UsersController : ControllerBase
{
    public UsersController(IUserService userService)
    {
        _userService = userService; // Injected automatically!
    }
}
