How Middleware Works

Each middleware can process requests and responses

// Custom Middleware
public class RequestTimingMiddleware
{
    private readonly RequestDelegate _next;
    
    public RequestTimingMiddleware(RequestDelegate next)
    {
        _next = next;
    }
    
    public async Task InvokeAsync(HttpContext context)
    {
        var stopwatch = Stopwatch.StartNew();
        
        await _next(context); // Call next middleware
        
        stopwatch.Stop();
        context.Response.Headers["X-Response-Time"] 
            = $"{stopwatch.ElapsedMilliseconds}ms";
    }
}

// Register in Program.cs
var app = builder.Build();
app.UseMiddleware<RequestTimingMiddleware>();
app.UseAuthentication();
app.UseAuthorization();
app.MapControllerRoute(...);
app.Run();
❓ Quiz: In what order should you register middleware?
Authorization → Authentication → Logging
Logging → Authentication → Authorization
Any order - it doesn't matter

The DI Container

Automatic dependency resolution and lifetime management

// Define services
public interface IUserService
{
    Task<User> GetByIdAsync(int id);
}

public class UserService : IUserService
{
    private readonly AppDbContext _context;
    
    public UserService(AppDbContext context)
    {
        _context = context;
    }
    
    public async Task<User> GetByIdAsync(int id)
        => await _context.Users.FindAsync(id);
}

// Register in Program.cs
builder.Services.AddScoped<IUserService, UserService>();

// Use in controller
public class UsersController : ControllerBase
{
    public UsersController(IUserService userService)
    {
        _userService = userService; // Injected automatically!
    }
}

Singleton

One instance for entire app

Scoped

One per request

Transient

New instance each time

✨ Pro Tip

Use Scoped for most services ( DbContext, UserService). Use Singleton for truly global things like ILogger.

Logging & Configuration

Debugging and environment settings
📝

Built-in Logging

// Constructor injection
private readonly ILogger<UserService> _logger;

public UserService(ILogger<UserService> logger)
{
    _logger = logger;
}

// Use it
_logger.LogInformation("Creating user {Email}", email);
_logger.LogWarning("User not found: {Id}", id);
_logger.LogError(ex, "Failed to process order");
⚙️

Configuration with appsettings.json

// appsettings.json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=localhost;Database=MyApp"
  },
  "Logging": {
    "LogLevel": {
      "Default": "Information",
      "Microsoft": "Warning"
    }
  }
}

// Use in code
var connectionString = builder.Configuration
    .GetConnectionString("DefaultConnection");
✨ Pro Tip

Use User Secrets in development (dotnet user-secrets set) for sensitive values. Never commit API keys or connection strings with passwords to source control!
