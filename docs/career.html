Project Checklist

What every portfolio project should include

  • Working CRUD operations (Create, Read, Update, Delete)
  • Database integration with EF Core
  • Clean API responses (proper status codes)
  • Basic authentication/authorization
  • Git version control with meaningful commits
  • README with setup instructions
  • Deployed to a live URL (Azure/AWSRender)
🐙

GitHub Best Practices

# Good commit messages
add user model and repository
implement user CRUD endpoints
add JWT authentication
fix validation error handling

# Branch naming
feature/user-authentication
bugfix/login-validation
hotfix/production-error
✨ Quality Over Quantity

Two well-documented, working projects are better than five half-finished ones. Focus on completing projects with clean code and proper architecture.

Common Questions

❓ Q: Explain the difference between GET and POST
GET retrieves data (safe, idempotent), POST creates data (not safe, not idempotent)
GET is faster than POST
POST is for reading data
They are the same thing
❓ Q: What is Entity Framework Core?
An ORM that maps C# objects to database tables
A JavaScript framework
A testing library
A CSS framework
❓ Q: Explain dependency injection in .NET
A technique where dependencies are provided to a class rather than created by it
A way to inject data into databases
A testing methodology
A type of database
💻

Coding Practice

  • Practice array/string manipulations
  • Understand Big-O notation basics
  • Know when to use List vs Dictionary
  • Be able to explain your solution
  • Test your code with edge cases
🎤

Behavioral Questions

❌ Weak Answer

"I'm a good team player."

✅ Strong Answer

"In my last project, I collaborated with two teammates to build a REST API. I took the lead on architecting the database schema while they handled the frontend integration."

💡 Use the STAR Method

Situation → Task → Action → Result. Structure your answers to show what you did and the outcome.
